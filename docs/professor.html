Tela do Professor

A página Teacher atua como uma interface inicial para professores, com um design minimalista e objetivo. Ela é composta por um cabeçalho e uma área de destaque onde um logotipo e uma mensagem motivacional são exibidos. O foco dessa página está em proporcionar uma visão clara e direta do propósito principal: gerenciar turmas e leituras. Essa simplicidade permite que os professores compreendam rapidamente o contexto da aplicação.

Já a página ClassDashboard é mais funcional e interativa, projetada para que os professores tenham controle sobre suas turmas. Nessa página, uma barra de busca permite filtrar turmas pelo nome, tornando a interface útil em cenários com grande quantidade de informações. Cada turma é representada por um cartão que fornece acesso direto às suas informações e funcionalidades, como visualização ou edição. Além disso, existe a possibilidade de criar novas turmas de forma rápida através de um botão dedicado.

Um ponto interessante na ClassDashboard é o uso de mensagens de feedback. Ao realizar ações em outras partes do sistema (como criar ou editar turmas), o sistema armazena mensagens temporárias que são exibidas ao professor ao retornar para essa página. Isso melhora a experiência do usuário ao garantir que suas ações foram concluídas com sucesso.

Além disso, a página utiliza atualizações dinâmicas para carregar os dados das turmas, fazendo requisições a uma API e garantindo que as informações exibidas estejam sempre atualizadas. Essa abordagem é complementada por componentes reutilizáveis, como o Input para a barra de busca e o ClassCard para representar cada turma, garantindo uma interface consistente e modular.

Exemplo do código

Uma parte interessante do código é o manuseio da barra de busca, que está diretamente ligada à funcionalidade de pesquisa por turmas na página ClassDashboard. Essa funcionalidade utiliza o estado local search para armazenar o texto digitado pelo usuário. Sempre que o conteúdo do campo de busca é alterado, a função associada ao evento onChange atualiza esse estado com o valor atual.

Em conjunto, o uso do estado search ativa o gancho useEffect. Esse gancho é configurado para ser executado toda vez que o valor de search muda. Dentro dele, uma requisição à API é feita, passando o nome da turma digitado pelo usuário como parâmetro (/classes?name=${search}). A resposta dessa requisição atualiza o estado classes, que armazena os dados das turmas correspondentes ao termo de busca.

Com isso, a interface reage automaticamente às alterações no campo de busca. Se o campo estiver vazio, um botão especial é exibido para adicionar uma nova turma. Quando o usuário digita algo, apenas as turmas que correspondem ao critério de busca são exibidas.
